## 1. 转换引擎：MarkItDown

项目使用微软 [MarkItDown](https://github.com/microsoft/markitdown)（MIT 许可，123K+ stars）作为唯一转换引擎，覆盖 14+ 格式。

```python
# converters/__init__.py — 核心入口
def convert_file(filepath: Path, output_dir: Path | None = None,
                 ocr_service=None, verbose: bool = False
) -> tuple[str, int, dict[str, str]]:
    """使用 MarkItDown 将文件转换为 Markdown，返回 (文本, 图片数, 图片映射)"""
    from markitdown import MarkItDown
    md = MarkItDown()
    result = md.convert(str(filepath))
    # ... 图片提取 + OCR 识别（可选，统一接口）
    return markdown, images_extracted, image_map
```

## 2. 图片提取

MarkItDown 将 PPTX/DOCX 中的图片转为内联引用。为支持后续的 OCR 识别，项目独立提取图片并建立映射：

| 格式 | 提取方式 | 图片目录 |
|------|---------|---------|
| PPTX | `python-pptx` 遍历 shape.image | `{filename}_images/` |
| DOCX | `zipfile` 解压 `word/media/` | `{filename}_images/` |

`_build_image_map()` 扫描 MarkItDown 实际输出的图片引用，按出现顺序与提取的图片路径一一匹配，构造映射 `{ref_name: actual_path}`。不依赖 MarkItDown 内部命名规则。

## 3. 图片识别（可选）

通过统一的 `OCRService` 接口（详见 [08-OCR统一管道设计](./08-OCR统一管道设计.md)）对提取的图片进行批量识别，结果以 blockquote 形式注入 Markdown：

```
![图片](path)

> 📷 **图片识别**: 这是一张架构图，展示了...
```

两种识别场景：
- **PPTX/DOCX 嵌入图片**：保留原图引用，追加 blockquote 描述（增强）
- **扫描型 PDF 整页**：替换全部内容为按页拼接的识别文本

底层 `CloudOCRService` 复用 `ImageFilter`（过滤纯色/过小/低分辨率图片）和 `ThreadPoolExecutor`（最多 5 并发）。

## 4. 支持的文件类型

```python
def get_supported_extensions() -> list[str]:
    return [
        ".pdf", ".docx", ".pptx", ".xlsx", ".xls",
        ".html", ".htm", ".epub", ".csv",
        ".txt", ".md",
        ".png", ".jpg", ".jpeg", ".gif", ".bmp", ".webp",
        ".mp3", ".wav", ".m4a",
        ".zip", ".msg", ".ipynb",
    ]
```

### 文件分类与处理策略

| 文件类型 | 示例 | 处理策略 |
|---------|------|---------|
| 可转换格式 | `.pdf`, `.docx`, `.pptx`, `.xlsx` | MarkItDown 转为 `.md` |
| 图片 | `.png`, `.jpg`, `.gif` | 直接复制到输出目录 |
| 其他 | `.mp4`, `.zip`, `.exe` | 复制 + 轻量 `.md` 元数据包装 |

## 5. 输出目录结构（目录 B）

```
目录 B/
├── project/
│   ├── 报告.pptx.md                ← MarkItDown 转换产物
│   ├── 报告.pptx_images/           ← 提取的图片
│   │   ├── slide1_img1.png
│   │   └── slide2_img1.jpg
│   ├── 视频.mp4                    ← 不支持格式：原文件复制
│   ├── 视频.mp4.md                 ← 不支持格式：元数据包装
│   └── 照片.jpg                    ← 图片：直接复制
```

每个 Markdown 文件带有 YAML frontmatter，记录来源路径、原始格式、转换状态等元数据。
