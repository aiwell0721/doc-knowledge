**更新日期**：2026-05-17

## 1. 整体架构

### 1.1 架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│                        Doc-Knowledge CLI                        │
├─────────────────┬──────────────────┬────────────┬───────────────┤
│ convert         │ extract          │ export     │ pipeline      │
│ (A→B)           │ (B→C)            │ (C→目标)   │ (A→B→C→导出)  │
├─────────────────┼──────────────────┼────────────┼───────────────┤
│ MarkItDown 封装 │ 语义去重         │ Obsidian   │ 一键完成      │
│ (14+ 格式)      │ SimHash 去重     │ MemoMind   │ 进度显示      │
│ 反向链接注入    │ 版本合并         │ Markdown   │ 错误恢复      │
│                 │ 价值评分         │            │               │
│                 │ 自动标签         │            │               │
│                 │ 增量更新         │            │               │
└─────────────────┴──────────────────┴────────────┴───────────────┘
```

### 1.2 转换层架构（MarkItDown 封装）

```
┌─────────────────────────────────────────────┐
│           Doc-Knowledge Convert             │
│                                             │
│  scan_dir() → detect_files()               │
│       ↓                                     │
│  MarkItDown.convert(filepath)  ← 微软开源    │
│       ↓                                     │
│  inject_backlinks(md, source_path)          │
│       ↓                                     │
│  write_output(mirror_dir)                   │
└─────────────────────────────────────────────┘
```

**策略**：转换逻辑复用 MarkItDown，自研差异化能力（去重/评分/标签/导出）。

## 2. 模块设计

### 2.1 转换层（MarkItDown 封装）

| 模块 | 职责 | 实现方式 | 依赖 |
|------|------|---------|------|
| `DocKnowledgeConverter` | 统一转换接口 | 封装 MarkItDown | markitdown>=0.1.0 |
| `BacklinkInjector` | 反向链接注入 | 自研 | — |
| `FileScanner` | 目录扫描 + 文件检测 | 自研 | pathlib |

**MarkItDown 已覆盖的格式**：
| 格式 | MarkItDown 转换器 | 状态 |
|------|------------------|------|
| PDF | pdfminer.six + pdfplumber | ✅ 开箱即用 |
| DOCX | mammoth + lxml | ✅ 开箱即用 |
| PPTX | python-pptx | ✅ 开箱即用 |
| XLSX | pandas + openpyxl | ✅ 开箱即用 |
| XLS | pandas + xlrd | ✅ 开箱即用 |
| HTML | markdownify | ✅ 开箱即用 |
| EPUB | 内建 | ✅ 开箱即用 |
| CSV | pandas | ✅ 开箱即用 |
| 纯文本 | charset-normalizer | ✅ 开箱即用 |
| 图片 | EXIF 提取 | ✅ 开箱即用 |
| 音频 | pydub + SpeechRecognition | ✅ 可选 |
| YouTube | youtube-transcript-api | ✅ 可选 |

**Doc-Knowledge 自研层**：
| 模块 | 功能 | 算法 | 阶段 |
|------|------|------|------|
| `Deduplicator` | TF-IDF 语义去重 | TF-IDF + 余弦相似度 | Phase 1 |
| `SimHashDeduplicator` | SimHash 大规模去重 | O(n) 汉明距离 | Phase 2 |
| `VersionMerger` | 版本合并 | 时间戳 + 内容长度评分 | Phase 2 |
| `ValueScorer` | 价值评分 | 启发式多因子评分 | Phase 1 |
| `Tagger` | 自动标签 | 关键词提取 + 同义词映射 | Phase 1 |
| `ObsidianExporter` | Obsidian 导出 | Markdown + YAML frontmatter | Phase 1 |
| `MemoMindExporter` | MemoMind HTTP 导出 | REST API 调用 | Phase 2 |
| `MemoMindMCPExporter` | MemoMind MCP 导出 | 直接写 SQLite | Phase 2 |

## 3. 数据流（Phase 2 增强）

```
目录 A (源文件)
    ↓ convert()
目录 B (MD镜像 + 反向链接)
    ↓ extract()
    │   ├── TF-IDF 去重 (默认)
    │   ├── SimHash 去重 (--simhash, 大规模)
    │   └── 版本合并 (--merge)
目录 C (去重后的知识文档)
    ↓ export()
目标 (Obsidian / MemoMind HTTP / MemoMind MCP本地 / Markdown)
```

## 4. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| CLI 框架 | click | 优雅、易扩展 |
| 进度显示 | rich | 美观、功能丰富 |
| **文件转换** | **MarkItDown** (微软, 123K stars) | **14+ 格式开箱即用，MIT 许可** |
| **DOCX 底层** | mammoth (MarkItDown 内置) | 语义转换，优于逐字提取 |
| **PDF 底层** | pdfminer.six + pdfplumber | 表格提取强 |
| **PPTX 底层** | python-pptx (MarkItDown 内置) | 唯一成熟选择 |
| **XLSX 底层** | pandas + openpyxl | DataFrame.to_markdown() |
| 去重/相似度 | 自建 TF-IDF + SimHash | 零外部依赖 |
| 文件检测 | pathlib | Python 标准库 |
