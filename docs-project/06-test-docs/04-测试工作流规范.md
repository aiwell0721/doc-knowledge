# Doc-Knowledge 测试工作流规范

**创建时间**：2026-05-17
**版本**：v0.1.0

---

## 一、TDD 工作流（规则 6）

### 开发流程

```
1. 写测试（应该失败）→ 2. 运行测试确认失败 → 3. 写最小生产代码 → 4. 运行测试确认通过 → 5. 重构（保持测试通过）
```

### 执行要求

| 场景 | 要求 |
|------|------|
| 新功能 | 必须先写 1+ 个失败测试，再写生产代码 |
| Bug 修复 | 必须先写复现测试，确认失败后修复 |
| 重构 | 先确认现有测试通过，重构后再次确认 |

### 检查清单

- [ ] 新增功能是否有对应测试？
- [ ] 测试是否在生产代码之前编写？
- [ ] 测试是否覆盖正常路径 + 错误路径？
- [ ] 测试名称是否描述场景（非实现细节）？

---

## 二、Bug 修复流程（规则 7）

### 强制步骤

```
1. 编写复现测试（应该失败）
2. 运行测试确认失败
3. 修复 Bug
4. 运行测试确认通过
5. 补充边界测试（如有必要）
6. 更新测试报告
```

### 禁止行为

- ❌ 直接修复 Bug，不写复现测试
- ❌ "手动验证"代替自动化测试
- ❌ 修复后不运行完整测试套件

---

## 三、覆盖率目标（规则 8）

### 目标

| 指标 | 目标 | 当前 |
|------|------|------|
| 总体覆盖率 | > 85% | 84% |
| CLI 覆盖率 | > 80% | 72% |
| 核心模块覆盖率 | > 90% | 86-100% |

### 未覆盖代码处理

| 覆盖率范围 | 处理 |
|-----------|------|
| < 70% | 必须补充测试 |
| 70-80% | 建议补充测试 |
| 80-90% | 评估是否合理 |
| > 90% | 可接受 |

### 运行命令

```bash
# 运行测试
make test

# 运行测试 + 覆盖率
make test-all

# 查看 HTML 覆盖率报告
open htmlcov/index.html
```
